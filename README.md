# Navbar

SaaS для онлайн-записи клиентов к бьюти/сервис-мастерам. Рынок — Узбекистан / СНГ.

> **Конституция проекта:** [CLAUDE.md](CLAUDE.md) — архитектурные правила и принятые решения (читается в начале каждой сессии).
> **Источник истины по страницам и фичам:** [app-spec.md](app-spec.md).

Ранние документы (исторический контекст, частично поглощены конституцией):
[docs/project-brief.md](docs/project-brief.md) · [docs/architecture.md](docs/architecture.md)

## Стек

**React + React Native (Expo), монорепо с общим TS-core.** Бэкенд — Supabase.
Целевая структура — `packages/core`, `apps/web`, `apps/mobile`, `supabase/` (см. [CLAUDE.md §1](CLAUDE.md)).
Flutter-прототип в `mobile/` НЕ основа продукта — оставлен как визуальный референс UX/фич.

## Структура репозитория (текущая)

```
supabase/ — схема БД, RLS-политики, тест изоляции (Wave 0 ✅)
web/      — Next.js: публичная страница записи navbar.uz/[slug] (ранний прототип)
mobile/   — Flutter-прототип (референс; продукт переезжает на React/RN)
docs/     — ранние документы
```

## Как запустить локально

Нужен установленный [Node.js](https://nodejs.org) версии 18 или новее (проверить: `node --version`).

```bash
# 1. Склонировать репозиторий (один раз)
git clone https://github.com/javadabdullaev97-coder/navbar.git
cd navbar/web

# 2. Установить зависимости (один раз)
npm install

# 3. Запустить
npm run dev
```

Открыть в браузере: <http://localhost:3000>

Демо-страница мастера: <http://localhost:3000/asror>

## Как поставить приложение на Android

APK собирается автоматически при каждом изменении в `mobile/`:

1. Открой [Releases](https://github.com/javadabdullaev97-coder/navbar/releases) → релиз **apk-latest**
2. Скачай `app-release.apk` на телефон
3. Открой файл — Android спросит разрешение на установку из неизвестных источников, разреши
