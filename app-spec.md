# app-spec.md — детальное описание приложения

Источник истины по страницам и фичам. `CLAUDE.md` держит архитектурные правила и ссылается сюда; здесь — вся детализация. Приложение двухролевое (мастер/клиент) + салонный контур; поверхности: web, приложение (iOS/Android), Telegram (Mini App + бот). Основной язык интерфейса — **English**; на старте также Russian и Uzbek.

---

## 1. Кросс-каттинг (действует во всём приложении)

### 1.1. Аккаунт, идентичность и мульти-профиль
- **Аутентификация по номеру телефона (OTP).** Один аккаунт = один номер.
- Под аккаунтом может быть **несколько профилей одновременно:** клиентский профиль и/или мастер (solo-организация) и/или членства в салонах (admin/master). Один человек по одному номеру может быть и клиентом, и мастером.
- **Логин по номеру → аккаунт находится → загружаются все его профили.** Если профиль один — вход прямо в него. Если несколько — вход в последний активный + тумблер.
- **Тумблер «Клиент ↔ Мастер/Админ»** на главном экране (в шапке) и в меню/профиле — переключение контекста **без выхода и повторного логина**.
- Если при логине человек выбирает роль, которой у него ещё нет (напр. был только мастером, выбирает «Клиент») — под тем же аккаунтом **создаётся новый профиль** (клиентский — с чистой историей), и тумблер становится доступен.
- Старый профиль при повторном логине той же ролью — **тот же самый**, со всей историей.

### 1.2. Смена устройства
- Данные живут на бэкенде (Supabase), не только на устройстве. Локальная база — лишь кэш.
- На новом устройстве: вход по номеру + OTP → **все профили, история, записи, клиенты подгружаются автоматически.** Отдельного «экспорт/восстановление» не требуется.

### 1.3. Языки (i18n)
- **База — English.** Поддержка на старте: English, Русский, Oʻzbek (uz-Latn).
- **Первый вход:** брать текущий язык устройства. Если это en/ru/uz — ставим его; если любой другой — по умолчанию **English**.
- Меняется в Настройках. Выбор языка **хранится на бэкенде** и следует за пользователем между устройствами.
- Все строки — через i18n-словари, ничего не хардкодить в UI.

### 1.4. Тема
- **По устройству (system light/dark)** при первом входе. Ручной оверрайд (Светлая / Тёмная / Системная) в Настройках, хранится на бэкенде.
- Акцент — по бренду мастера (см. дизайн-токены), красится только акцент.

### 1.5. Ввод телефона
- Международное поле. Плейсхолдер по умолчанию **+998**.
- По введённому коду **определяется страна и показывается флаг** (напр. ввёл +7 777 → флаг Казахстана; +996 → Кыргызстан и т.д.).
- Валидация и форматирование по libphonenumber; хранение в формате E.164.

### 1.6. Синхронизация с календарём (выборочная)
- Мастер/админ может **подключить внешний календарь** (Google / Apple).
- **Выборочно:** выбрать целевой календарь и что синкать (все брони / только подтверждённые / по услугам). Тумблер вкл/выкл.
- MVP — односторонний пуш брони → в календарь (как события с напоминаниями). Опционально позже — двусторонний (читать «занятость» из внешнего календаря, чтобы не было двойных броней).

### 1.7. Уведомления
- Каналы: push (приложение), Telegram-бот, in-app. События: новая бронь, подтверждение/отмена/перенос, напоминание, сообщение в чате, заявка на вступление в салон.

## 2. Клиент — страницы и фичи

- **Онбординг:** выбор «Клиент» → телефон + OTP → имя → язык/тема унаследованы от устройства.
- **Поиск / Дискавери:** список мастеров + карта; фильтры (услуга, район, цена, рейтинг); поиск по имени/услуге. Показываются только мастера с `visible_in_search = true`.
- **Профиль мастера:** обложка (бренд-цвет/фото), имя, рейтинг, район; табы — Услуги, Портфолио (галерея), Отзывы, Инфо (адрес, карта, часы, контакты, соцсети).
- **Флоу записи:** услуга → дата → время → (депозит, если включён) → подтверждение. Публичная запись по ссылке/QR работает всегда, даже без входа.
- **Мои записи:** предстоящие / прошлые; статусы (подтверждено, ожидает, отменено, завершено); отмена и перенос без звонка (с уведомлением мастеру); история.
- **Мои мастера / Избранное:** сохранённые мастера, быстрый повтор записи.
- **Чат:** переписка с мастером/салоном (Supabase Realtime).
- **Уведомления:** центр уведомлений.
- **Профиль и настройки:** учётная запись, язык, тема, уведомления, приватность, удаление аккаунта, выход. **Тумблер на интерфейс мастера** — если у аккаунта есть мастер-профиль.
- **Нижняя панель (клиент):** Поиск · Мои записи · Избранное · Профиль.

## 3. Мастер (соло) — страницы и фичи

- **Онбординг:** выбор «Мастер» → телефон + OTP → визард: специализация → профиль/«о себе» → места работы → услуги → график.
- **Дашборд (Главная):** приветствие, метрики дня (записей, загрузка, доход), следующий клиент, лента «сегодня/дальше», быстрые действия.
- **Расписание:** день/неделя, слоты, брони, перерывы, блокировки времени; создание/перенос/отмена брони; синхронизация с внешним календарём (§1.6).
- **Записи:** карточка брони (услуга, клиент, время, депозит, заметки, статус).
- **Услуги:** редактор услуг (название, длительность, цена, категория, вкл/выкл, «новые услуги»).
- **Доступность:** рабочие часы, выходные, смены.
- **Клиенты (CRM):** база клиентов, история посещений, заметки, дни рождения, группы, скидки (базовые/персональные/по типу); импорт из адресной книги.
- **Финансы:** учёт доходов и расходов, аналитика (по услугам/клиентам/периодам).
- **Портфолио:** галерея работ (отдельно от обложки).
- **Отзывы:** список, ответы, шеринг отзыва.
- **Публичная страница записи (настройки):** обложка (цвет бренда из палитры / фото), `visible_in_search` (по умолчанию выкл), ссылка/QR для клиентов.
- **Платежи/депозиты:** подключение локального эквайера (Payme/Click), депозит-за-бронь, история платежей.
- **Чат, уведомления, тарифы/подписка** (для соло), профиль и настройки (язык, тема, аккаунт).
- **Нижняя панель (мастер):** Главная · Записи · Клиенты · Профиль. **Тумблер на интерфейс клиента** — если у аккаунта есть клиентский профиль.

## 4. Салон — страницы и фичи

### 4.1. Регистрация
Выбор «Салон» → затем **Админ** или **Мастер**:
- **Админ** → создаёт салон (owner/admin новой организации) или принимает приглашение админом существующего.
- **Мастер** → присоединяется к существующему салону (код/ссылка/поиск) → заявка (`pending`) → админ подтверждает.

### 4.2. Админ (администрация салона) — все поверхности, десктоп-веб приоритетен
- **Дашборд салона:** сводка по всем мастерам (записи, загрузка, доход).
- **Управление мастерами:** приглашения и заявки (approve/reject), список мастеров, роли, отвязка; занятость мест подписки (seats).
- **Сводное расписание:** календарь по всем мастерам сразу; переход в расписание конкретного мастера.
- **Услуги и прайс салона.**
- **Профиль/презентация салона:** сайт-визитка, обложка, адрес, часы, портфолио.
- **Финансы:** по каждому мастеру + агрегат; аналитика.
- **Подписка и биллинг:** план, места (seats), оплата — на уровне организации.
- **Коммуникации:** SMS/уведомления, групповые рассылки (акции, дни рождения).
- **Отзывы, настройки, роли/доступы.**

### 4.3. Мастер в салоне
- Тот же мастерский инструментарий, но в контуре салона; админ видит его расписание и брони (RLS «org+роль»).
- Присоединение — через заявку/приглашение с подтверждением.
- Открытые решения (см. §6): может ли мастер быть ещё и соло / в нескольких салонах; чьи клиенты при уходе.

## 5. Общие экраны и компоненты

- Экран входа (телефон + OTP), международный телефон-инпут с флагом (§1.5), выбор языка, переключатель темы.
- Тумблер ролей (Клиент ↔ Мастер/Админ); при нескольких организациях — вторичный выбор организации.
- Центр уведомлений, чат, пустые состояния, экран удаления аккаунта.

---

## 5a. Порядок волн — что отложено до «волны интеграций»

Решение владельца: сначала строим продукт (фичи, экраны, логика, UI/UX) на живом
бэкенде с **временной авторизацией** (email-ссылка для мастера, публичная запись
без входа для клиента). Всё «внешнее» переносится в отдельную **волну интеграций**
в конце — оно требует юрлица, договоров и денег:

- **SMS-OTP** (Eskiz, ~95 сум/шт) + автоподстановка кода (iOS oneTimeCode / Android
  SMS Retriever). Заменит временный email-вход. Стоимость OTP-входа мастера —
  копейки; объёмная статья — напоминания клиентам → включаются в тариф Про квотой,
  метринг-кошелёк позже.
- **Платежи** (Payme / Click / Uzum) — подписка мастера, депозиты.
- **Telegram** — бот-уведомления + Mini App.

До этой волны их НЕ трогаем; строим всё остальное.

## 6. Открытые решения (трекаются здесь, дефолты в CLAUDE.md)

- Мультичленство: мастер в >1 салоне и/или одновременно соло? Дефолт — схема M:N, MVP одна активная организация на мастер-контекст.
- Клиенты при уходе мастера из салона — чьи. Дефолт — принадлежат мастеру (переносимы), салон видит брони/расписание.
- Что именно видит админ по мастеру (расписание/брони/выручка; личные заметки приватны).
- Ценообразование подписки салона (по местам).
- Календарь: только пуш или двусторонняя синхронизация.
